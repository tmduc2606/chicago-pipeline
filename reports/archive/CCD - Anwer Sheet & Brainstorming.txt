Chicago Crime Dashboard — UI/UX Design Rubric | Answer Sheet
1. Dashboard Layout & Navigation
- Decision
+ Sidebar behavior: Collapsible (icons on tablet, full on desktop)
+ Sidebar position: Left (standard for data dashboards)
+ Top bar contents: Global filters + page title
+ Content grid: Fluid with max-width 1440px
+ Page transitions: Fade (200ms, standard easing)
+ Footer: Minimal — pipeline status indicator + last updated
2. Information Architecture — 8 Pages
- Questions:
+ Should Overview be the landing page (/) or should there be a separate landing? -> Separate Landing (avoid redundancy spaces if creating a new page)
+ Should Pipeline and Quality be public or behind a /admin route? -> Behind a /admin route
+ Do you want a 9th page for "About" / "Data Sources" / "Methodology"? -> Yes
3. Chart Types & Visualization Matrix
- Questions:
+ Do you want sparklines inside KPI cards or just the big number? -> Sparklines inside KPI cards 
+ Should the heatmap use ECharts (more features) or Recharts (simpler, consistent)? -> ECharts (more features)
+ Do you want a "View as table" toggle on every chart? -> Probably, though i'm not so sure how it works
4. Global Filter System
- Questions:
+ Should filters be in the top bar (always visible) or a slide-out panel? -> in the top bar (always visible)
+ Do you want filter chips/badges showing active filters? -> Yes
+ Should the URL be shareable (public filters) or session-only? -> Session-only
5. Loading, Empty, and Error States
- Questions:
+ Do you want skeleton loaders or simple spinners? -> Skeleton loaders
+ Should error states show technical details in dev mode only? -> Probably, though i'm not so sure about the complexity of the message prompt.
+ Do you want a global "connection status" indicator (green/red dot)? -> Yes
6. Mobile Responsiveness
- Questions:
+ Do you want a dedicated mobile layout or just responsive scaling? -> Dedicated mobile layout
+ Should maps be full-screen on mobile with a back button? -> Yes
+ Do you want swipe gestures between pages on mobile? -> Yes
7. Accessibility (WCAG 2.2 AA)
- Questions:
+ Do you want a "high contrast mode" toggle? -> Maybe
+ Should every chart have a "View as table" button? -> This depends on the level of feature importance
+ Do you want to support screen reader announcements for live data updates? -> Definitely, yes
8. Performance Budget (The answers may not fully reflect my correct decisions)
- Questions:
+ Do you want bundle analysis on every build (rollup-plugin-visualizer)? -> Maybe
+ Should we prefetch all pages on initial load or only on hover? -> Yes, i think
+ Do you want performance monitoring in production (Web Vitals)? -> Yes, important
10. Color Palette Validation
- Questions:
+ Do you want to test the categorical palette against colorblind simulators? -> Yes
+ Should we add patterns (hatching, dots) to charts for colorblind users? -> Yes
+ Do you want a dark mode / light mode toggle, or dark-only? -> dark mode / light mode toggle
